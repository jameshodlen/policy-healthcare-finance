Methodology

Data Source

All data is extracted from machine-readable files (MRFs) that hospitals are required to publish under the CMS Hospital Price Transparency Rule (45 CFR 180.50). Since January 1, 2024, hospitals must maintain a cms-hpt.txt discovery file at their website root, pointing to downloadable chargemaster files in CMS-standardized JSON or CSV format.

Discovery Process

We query the CMS Hospital General Information dataset to identify all registered hospitals in each state. We then fetch cms-hpt.txt from each health system's website, extract MRF download URLs, and download the files with rate limiting (2-second delay between requests). For systems where the TXT file is not at the standard location, we fall back to scraping the hospital's price transparency webpage for direct file links.

Transparency Score

The transparency percentage measures the share of data rows containing a payer-specific negotiated rate. This includes negotiated dollar amounts, negotiated percentages, and estimated allowed amounts. Rows containing only gross charges, cash prices, or de-identified min/max amounts are not counted as transparent.

For files in CSV "tall" format (v3.0.0), each row represents one item–payer combination. We count rows with a non-zero standard_charge|negotiated_dollar value.

For files in CSV "wide" format (v2.0.0), payer-specific rates are embedded in column headers. We count rows where at least one payer column has a non-zero value.

For JSON files, we count items in standard_charge_information that contain at least one payers_information entry with a negotiated rate.

Cash Discount Analysis

We compute the cash discount by comparing the average discounted_cash price to the average gross_charge across a sample of items. A "mechanical" discount means the ratio is consistent — the hospital applies a flat percentage formula. Cash discount coverage measures what percentage of data rows include a cash price at all.

Limitations

Coverage. We analyze files we can access. Some health systems block automated download via TLS configuration, web application firewalls, or robots.txt. SSM Health (Wisconsin/Illinois), Mayo Clinic Health System community hospitals, and CentraCare (Minnesota) were inaccessible during this harvest.

Truncation. Files exceeding 200MB were truncated during download. Profiling metrics are computed from the available data and may undercount total items for very large files.

Schema variation. Hospitals use different CMS schema versions (2.0.0, 2.2.0, 3.0.0) and different file structures (tall, wide, JSON). Our profiler handles all formats but may undercount transparency for unusual structures — as occurred initially with UW Health's wide-format file and UChicago's split structure.

Point-in-time. This analysis reflects files harvested on March 24, 2026. Hospitals update their files periodically — some annually, some more frequently. Rankings may change with each update cycle.

Reproducibility

Every metric on this platform is programmatically computed from the published CMS files. The harvest, parsing, and profiling pipeline is documented and deterministic. Hospital inventory data is sourced from the CMS Hospital General Information API. All source URLs are logged in the harvest manifest.
